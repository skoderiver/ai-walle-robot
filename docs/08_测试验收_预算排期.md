# 测试、验收、预算与排期

## 1. 开发阶段建议

### 第 1 阶段：2 周

- 明确尺寸、目标功能、预算
- 建立比例模型
- 确定主控与底盘路线

### 第 2 阶段：2 到 4 周

- 打通底盘驱动
- 打通头部两自由度
- 完成基本串口协议

### 第 3 阶段：2 到 4 周

- 完成正式结构件
- 完成线束、电池、电控布局
- 形成可展示样机

### 第 4 阶段：2 到 6 周

- 做喷涂和外观细化
- 做展示动作与遥控界面
- 做视觉/语音升级

## 2. 测试清单

### 机械测试

- 履带张紧测试
- 重心稳定测试
- 螺丝松动测试
- 关节极限位测试

### 电气测试

- 空载功耗
- 峰值电流
- 电压跌落
- 连续运行温升
- 急停有效性

### 软件测试

- 心跳丢失停车
- 状态切换正确性
- 手动接管自动模式
- 断电重启恢复

### 展示测试

- 现场灯光条件下视觉效果
- 音效音量
- 网络不稳定时是否还能本地演示
- 一键演示脚本是否可重复执行

## 3. 验收表

### 基础能力

- 可移动
- 可转头
- 可遥控
- 可急停
- 可连续展示

### 展示效果

- 远距离外观识别度高
- 近距离细节可看
- 动作有角色感
- 现场维护难度可接受

## 4. 风险清单

- 外壳做得太早，导致后续改结构很痛
- 头部过重，舵机持续发热
- 电源设计不足，导致频繁复位
- 履带摩擦过大，导致驱动效率低
- AI 模块过早接入，拖慢整体进度

## 5. 推荐里程碑

1. `M1`：底盘能动
2. `M2`：头能动
3. `M3`：Unity 控制台能控实体
4. `M4`：整机外观成型
5. `M5`：完成展陈演示
6. `M6`：Jetson/ROS2/视觉升级

## 6. 建议你本周就做的事

1. 决定整机尺寸
2. 选定 V1 功能范围
3. 先买一套底盘验证件
4. 在 Unity 里建数字样机
5. 画出供电框图和控制框图
