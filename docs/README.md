## 1. 项目定位

本项目当前目标已经升级为：做一台具备 `自然对话 + 真实环境感知 + 室内自主移动` 能力的瓦力风格机器人。视觉风格仍参考公众熟悉的 WALL-E 形象与英伟达展陈里常见的 AI 机器人展示方式，但实施路线不再以“遥控展陈”为核心，而是以 `监督式自主机器人` 为核心。

建议先做 `V1 平地自主版`，再升级到 `V2 多房间语义导航版`，最后再评估 `V3 跨楼层版`：

- `V1 平地自主版`：语音输入输出、平地建图导航、头部动作、灯光音效、急停保护。
- `V2 多房间语义导航版`：语义地点、自动避障、任务状态反馈、失败恢复、表情状态机。
- `V3 跨楼层版`：电梯联动或特种楼梯平台，跨楼层任务代理，完整安全策略。

## 3. 推荐实施顺序

1. 先读 [10_自主导航与大模型可行性.md](./10_自主导航与大模型可行性.md)，明确“平地自主”和“跨楼层”不是同一难度级别。
2. 再读 [01_功能设计.md](./01_功能设计.md)，重新锁定目标版本。
3. 再读 [02_技术方案选型.md](./02_技术方案选型.md)，默认采用 `ESP32 + Jetson + ROS2 + 大模型代理` 架构。
4. 按 [03_机械与建模方案.md](./03_机械与建模方案.md) 先完成数字样机和尺寸冻结。
5. 按 [04_硬件选材与BOM.md](./04_硬件选材与BOM.md) 采购。
6. 按 [05_电气与控制系统.md](./05_电气与控制系统.md)、[06_组装流程.md](./06_组装流程.md)、[07_软件架构_含Unity.md](./07_软件架构_含Unity.md) 联调。
7. 最后用 [08_测试验收_预算排期.md](./08_测试验收_预算排期.md) 做分阶段验收。

## 4. 设计原则

- 优先保证安全与可靠运动，再保证感知导航，最后再叠加对话体验。
- 把“大模型理解”和“运动控制”彻底分层：高层用 Linux/Jetson，底层用 MCU。
- 尽量模块化：头部、躯干、手臂、底盘、电池盒、电控盒可拆卸。
- 先验证自由度和重心，再开始外壳精修。
- 先做有线调试版本，再封装成无线自主版本。
- 跨楼层能力默认视为高风险需求，除非明确选用电梯或特种楼梯底盘。

## 5. 你作为 Unity 开发工程师的优势

- 可以优先建立 `数字孪生 + 状态机 + 任务代理调试台`，再反推实体硬件接口。
- 可以先在 Unity 里验证语义地点、机器人状态、任务执行可视化，再接实体底盘。
- 接入 ROS2 后，Unity 会非常适合做任务流程仿真、导航状态监控与演示控制。

## 6. 一句话结论

目前最稳妥的落地路线是：

`Fusion 360 / Blender 建模 + 3D 打印外壳 + 平地自主底盘 + ESP32 底层控制 + Jetson 高层导航与感知 + ROS2/Nav2 + OpenAI Responses API / Realtime API 做任务代理与语音交互 + Unity 做数字孪生与调试界面`
