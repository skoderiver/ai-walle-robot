## 1. 项目定位

本项目默认目标是做一台“展览展示级”的瓦力机器人，视觉风格参考公众熟悉的 WALL-E 形象与英伟达展陈里常见的 AI 机器人展示方式，但实施路线以个人可 DIY、可维护、可逐步升级为前提。

建议先做 `V1 展陈版`，再升级到 `V2 互动版`，最后再做 `V3 AI 版`：

- `V1 展陈版`：头部转动、眼睛动画、履带底盘移动、灯光音效、遥控演示。
- `V2 互动版`：加入语音唤醒、简单视觉跟随、自动巡航、表情状态机。
- `V3 AI 版`：Jetson 本地推理、ROS2、多模态交互、Unity/Isaac Sim 联调。

## 4. 设计原则

- 优先保证外观神韵，其次是可靠运动，最后才是 AI 能力。
- 把“大计算”与“实时控制”分层：高层用 Linux/Jetson，底层用 MCU。
- 尽量模块化：头部、躯干、手臂、底盘、电池盒、电控盒可拆卸。
- 先验证自由度和重心，再开始外壳精修。
- 先做有线调试版本，再封装成无线展陈版本。

## 5. 你作为 Unity 开发工程师的优势

- 可以优先建立 `数字孪生 + 状态机 + 遥控 UI`，再反推实体硬件接口。
- 可以把眼睛动画、头部运动、表情切换先在 Unity 中验证，再映射到舵机角度与屏幕素材。
- 后续若接入 ROS2，可直接做仿真、调参与录制演示脚本。

## 6. 一句话结论

最稳妥的落地路线是：

`Fusion 360 / Blender 建模 + 3D 打印外壳 + 铝型材/亚克力骨架 + 双履带减速电机 + ESP32 底层控制 + 树莓派或 Jetson 做高层交互 + Unity 做数字孪生与调试界面`
