# 功能设计

![自主能力栈](./assets/autonomy_stack.svg)

## 1. 产品目标

目标不是先做“什么都能做”的机器人，而是先做一台：

- 一眼能认出是瓦力风格
- 能在真实室内环境稳定自主移动
- 能和人自然对话
- 后续可扩展跨房间与跨楼层任务的机器人

## 2. 功能分级

### P0 必做

- 语音输入
- 语音输出
- 平地自主导航到预设点位
- 障碍物检测与避障
- 机器人状态播报
- 头部左右转动
- 头部俯仰
- 眼睛发光或眼睛屏幕动画
- 急停保护
- 失联停车
- 电量不足返航或停靠

### P1 推荐

- 语义地点管理
- “来找我”“回充电站”“去门口”等任务语义
- 手臂抬起/摆动
- 电量显示
- 自动回充或回待机位
- 任务失败后的语音说明

### P2 进阶

- 摄像头人脸/人体检测
- 视觉跟随
- 云端或本地 LLM 对话代理
- 语义地图
- 电梯联动跨楼层
- 特种底盘楼梯通过
- Unity 仿真与状态监控

## 3. 场景拆分

### 场景 A：待机

- 眼睛缓慢扫视
- 头部偶尔微动
- 播放低频机械音
- 灯光呼吸

### 场景 B：自然语言任务执行

- 用户说“去客厅”
- 机器人确认目标
- 自主导航并口头播报状态
- 到达后反馈完成

### 场景 C：对话互动

- 用户提问
- 机器人调用大模型回答
- 机器人根据当前状态决定是否执行物理任务

## 4. 推荐自由度

如果是第一次做自主机器人，建议姿态自由度仍保持克制：

- 底盘：左右履带 2 通道差速
- 头部左右：1 DOF
- 头部俯仰：1 DOF
- 左手臂：1 DOF
- 右手臂：1 DOF

合计推荐：`4 个姿态自由度 + 1 套履带驱动`

## 5. 交互模式设计

### 模式 1：语音任务

- 用户说出目标
- 高层代理解析意图
- 导航层执行任务

### 模式 2：语义导航

- “去客厅”“回去充电”
- 目标映射到地图点位
- 支持失败重试和拒绝执行

### 模式 3：监督式自主

- 自动感知人或障碍
- 自动避障与重规划
- 但保留急停与人工监督权
- 任意时刻可以急停

## 6. 不建议第一版就做的事情

- 双足行走
- 复杂机械手抓取
- 全车大量自由度联动
- 完全无监督地在未知环境自由乱跑
- 一上来就做全金属重型结构
- 一上来就做“瓦力外观 + 自主爬楼梯”

## 7. 验收标准

### V1 验收

- 能识别并响应语音指令
- 能在单层平地到达 3 个以上预设点位
- 能检测障碍并安全停下或绕行
- 连续运行 20 分钟不失控

### V2 验收

- 可以稳定执行多房间导航
- 任务失败时能口头说明原因
- 语音、视觉、导航模块能稳定跑 30 分钟

### V3 验收

- 可实现跨楼层任务
- 楼梯或电梯策略安全可控
- Unity 与实体状态同步
