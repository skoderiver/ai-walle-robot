# 功能设计

## 1. 产品目标

目标不是先做“全自动 AI 机器人”，而是先做一台：

- 一眼能认出是瓦力风格
- 能稳定移动和转头
- 现场展示效果好
- 后续可扩展语音、视觉、AI 的机器人

## 2. 功能分级

### P0 必做

- 履带前进、后退、原地转向
- 头部左右转动
- 头部俯仰
- 眼睛发光或眼睛屏幕动画
- 基础音效播放
- 遥控模式
- 急停保护

### P1 推荐

- 手臂抬起/摆动
- 脖颈伸缩或缓冲结构
- 电量显示
- 手机/Web 控制面板
- 展示动作脚本
- 障碍物检测

### P2 进阶

- 麦克风 + 关键词唤醒
- 摄像头人脸/人体检测
- 视觉跟随
- 自动巡航
- 云端或本地 LLM 对话
- Unity 数字孪生监控

## 3. 场景拆分

### 场景 A：展陈静态待机

- 眼睛缓慢扫视
- 头部偶尔微动
- 播放低频机械音
- 灯光呼吸

### 场景 B：人工遥控演示

- 主讲人手动控制移动
- 一键触发“打招呼”“发现物体”“兴奋”等动作
- 配合语音或背景音乐演示

### 场景 C：半自动互动

- 看到人后头部朝向目标
- 近距离停止
- 播放预设语音与动作

## 4. 推荐自由度

如果是第一次 DIY，建议控制在以下自由度：

- 底盘：左右履带 2 通道差速
- 头部左右：1 DOF
- 头部俯仰：1 DOF
- 左手臂：1 DOF
- 右手臂：1 DOF

合计推荐：`4 个姿态自由度 + 1 套履带驱动`

## 5. 交互模式设计

### 模式 1：纯遥控

适合最先落地。

- 游戏手柄 / 手机网页 / Unity 控制台发送命令
- 所有运动由人实时接管

### 模式 2：动作脚本

适合展会与视频拍摄。

- 定义若干动作序列
- 一键触发
- 保证动作可重复、稳定

### 模式 3：监督式自动

适合中后期升级。

- 自动感知人或障碍
- 但保留人工接管权
- 任意时刻可以急停

## 6. 不建议第一版就做的事情

- 双足行走
- 复杂机械手抓取
- 全车大量自由度联动
- 纯语音控制且无遥控兜底
- 一上来就做全金属重型结构

## 7. 验收标准

### V1 验收

- 平地连续运行 20 分钟不掉链、不失控
- 头部运动无明显抖动
- 遥控响应延迟在可接受范围
- 外观完成度达到展示级

### V2 验收

- 可以稳定执行 3 到 5 个展示动作脚本
- 可检测前方障碍并减速/停止
- 语音或视觉模块能稳定跑 30 分钟

### V3 验收

- Jetson/高层计算稳定运行
- Unity 与实体状态同步
- 能完成半自动互动演示
